# Security Policy

## Reporting a Vulnerability

Report suspected vulnerabilities privately with GitHub Private Vulnerability Reporting from this repository's Security tab. Include the affected component, impact, reproduction steps, and a minimal synthetic example when possible.

Do not include secrets, credentials, private hostnames, production data, or exploit details in a public issue. Public issues may be used for non-sensitive security hardening only after confirming that they disclose no vulnerability or private data.

The maintainers will acknowledge a private report, investigate it, and coordinate disclosure and remediation through the private report. Response and release timing depend on severity, reproducibility, and maintainer availability.

## Supported Versions

Security fixes are made on the current development line until versioned release support is documented here. Operators should run a reviewed current revision and apply published security updates promptly.
